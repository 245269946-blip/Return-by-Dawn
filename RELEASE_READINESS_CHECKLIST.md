## ❌ 未就绪（非美术、非旋律——真实缺口）

### 1. 游戏外壳 / 前端流程（ENGINE 层，非美术）
- **无标题页 / 主菜单**（开始 / 继续 / 设置）：当前 `Main._ready` 直接进序章，无菜单状态机。
- **无暂停菜单**（继续 / 设置 / 回主菜单）。
- **无设置页**（主音量 / 静音）：`AudioManager` 音量为硬编码，无用户可调接口。
- **无制作名单 / 关于页**。
- 说明：菜单的*视觉样式*属美术（已排除）；但菜单*存在与否 / 状态机 / 设置逻辑*是引擎脚手架，当前缺失。

### 2. v6 §八「图书馆作为角色」动态探索系统（设计了，引擎未落地）
- 书架每日微移位、雨天限定书、管理员也会收到通知——这三条 v6 明确写的"探索欲"机制，**引擎中无实现**（仅 `void_room` 第9空间按 v6 正确设为"永不开启"）。
- 影响：v6 设计用以补"情绪线结束后中间动力不足"，属内容/机制层缺口，非美术。

### 3. 实际导出构建包（packaging）
- `export_presets.cfg` 在，但 `build/win/*.exe` 与 `build/web/index.html` **未真正导出**。
- 要上 TapTap / Web 还需跑一次 export（且导出时建议接入图标、版本号）。

### 4.（建议）后段夜文案逐字保真走查
- 夜G/H/I 体量偏轻（13/11/10/11 KB）符合 v6「第三幕狂欢每夜 5–8 分钟、结构可重复多夜、写 2–3 个示意」的设定，**非缺陷**。
- 但本次仅对 终章/夜E/夜F 做了关键字保真核验（均通过）；夜G/H/I 与 夜D 的逐句贴合 v6 节拍**未逐字通读**。建议上线前做一轮内容 fidelity 走查（低风险，结构+点测已通过）。

### 5. 次要
- 应用图标、版本号字符串未配置；无障碍（字号/字幕/色弱）未做（独立游戏可选）。

## 结论
**不是"全部就绪"。** 叙事主体 + 引擎 + 非旋律音频 + 红线 + 存档 + 发布预设均已就绪并通过验证；但作为"合格独立游戏"的正式版，仍缺：
1. 游戏外壳（标题/菜单/设置/暂停/制作名单）
2. v6 §八 动态探索系统
3. 实际导出构建
4. （建议）后段夜文案保真走查
