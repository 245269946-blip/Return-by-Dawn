## 〇、先厘清一个命名歧义
- 此前已定的 **断裂原型 B「自我版本遗弃」** ≠ 本轮的 **B 行动型**。
- 本轮「B 行动型」= 循环「正确答案」的落点选型：玩家最终不是"想通了"，而是**在循环里亲手做出当年没做的事**（偏甲·门没锁你走出）。这把鬼灭"命运重发同一命题直到答对"的发动机，落成一个可操作的动作，而非纯认知。

## 一、把用户的 1/2 翻译成设计语言

### 1.1 逃避 = 核心情感引擎（不是主题词，是机制）
用户原话拆成可落地的三层：
- **每一次逃避都"当下有效"**：逃避的诱饵是即时止痛——你躲开了尴尬/责任/痛，那一刻确实轻松了。这正是它成瘾、循环能成立的原因。
- **但每一次逃避，你就把自己"对（倒）下了"**：逃避不是免费的，它把"某个版本的自己"留在了那一刻（= 已锁脊柱"遗落最久的版本"的机制成因）。一次小逃避 → 一个遗落碎片 → 长期累积 → §2.1 的"自我早已破碎"是**结果**，不是前提。
- **"对下"是复利式的**：越逃越难回头，因为回头要面对的是所有累积的"没做"。

→ 设计含义：**不要在前期宣告"你烂透了"，要让玩家在每一轮的"当下有效"里，亲眼看自己一点点把版本留在原地。** 破碎是玩家自己数出来的，不是作者告诉的。

### 1.2 轮回 = 夜循环；图书馆的出现条件（17:24 修正）
- 轮回 = 已锁的「固定流程拍子」(通知→图书馆→探索→知被遗忘→三选一) 的反复。每夜是同一张考卷重发。
- **图书馆为何出现（修正后的触发逻辑）**：当你在现实里遇到一桩**和"你已经逃避过"高度相似的处境**时，图书馆作为那张重发的通知单出现——不是新鲜的崩溃，而是**已逃命题的回声/重演**（= 兔兔岛同构，已锁）。它之所以一次次回来，正因为那件事你从没真正面对过，只逃过。
- **管理员在其中的角色**：他只是**提醒你**（"这封通知，你又收到了"），但剧情流程上**强引导你继续逃避**——每夜最舒服、最自然的那条路，就是把书还了/陪他玩了/又躲过去了。这条"顺手就逃"的引导，正是为结尾 BE 蓄力的。
- **与已锁纪律的对账**：强引导 ≠ 惩罚。游戏仍**无惩罚失败**、**柔性触发**（没有强制"你现在必须知道X"的硬节点）。所谓强引导，是 UX 把"逃避"做成最温暖、最不费力的默认路径——温柔的陷阱，不是审判。

### 1.3 管理员 = 没逃的那个你（铁驭隐喻）
- 管理员 = 逾期最久的那本书 = **你遗落的版本里"留下来面对的那一个"**（已锁脊柱，现被用户点得更亮：他不是"过去的你"泛指，而是"当年没逃、留下扛的那部分你"）。
- **铁驭（Titanfall 2 BT）双属性**（用户给的极好意象）：
  - ① **独自面对逃避的挣扎与无力**：他留在馆里，一遍遍替你收着所有遗落碎片，却**无法替你不再逃避**——他的无力感来自"我什么都做了，唯独不能替你迈步"。
  - ② **羁绊带来的分别**：正因为他和你绑得深，终章的"放你走"才是真痛。BT 把命交给 Cooper，管理员把"Loop 的钥匙"交还你——**因为他爱你，所以他要你离开他。**
- 这把 §12.2 收束句「灯还亮着…你得自己往前走」从"温柔放行"升级成"最痛的成全"。

### 1.4 循环本身 = 终极逃避；结尾须拆成三拍（17:24 修正）
用户原话："你仍然能继续逃避回到图书馆和他一起，但他更想你抛下他走出阴影走向现实。"

结尾不是一拍揭晓带过，而是**两件独立的事 + 一个过渡夜**，顺序固定：

- **第一拍 · 揭晓"管理员即是自己"**：你终于看清，陪你来回的、是当年没逃、留下来的那个你（= 已锁脊柱）。这一拍只回答"他是谁"，不回答"你该怎么办"。
- **第二拍 · 管理员和你的"逃避开心的夜"（新增过渡）**：身份认出之后、分离之前，特写一夜——你们像两个终于相认的老友，在馆里**开心地逃避**：还书、闲聊、玩、把现实挡在门外。这一夜越甜，越为最终分离蓄力；它证明你们之间的羁绊是真的，所以"走"才是真痛，不是解脱。
- **第三拍 · 揭晓"逃避真相" + 抉择分离**：现在才把第二层掀开——你发现你最贪恋的"回来找他/还书/陪他玩"本身，就是**最后一道逃避**；图书馆是你用来躲现实的温柔壳。抉择=你愿不愿意不再回来。门没锁（甲意象），他希望你走出去，最难的动作是转身。

**管理员的行为边界（17:24 写死纪律）**：他**不会强提示你该怎么做**——他做不了"替你迈步"的事。他唯一能主动做的只有两件：
1. **陪伴**（留灯、等你、陪你玩）；
2. **寄出那封逾期通知**（= 把你一次次唤回图书馆的，正是他寄出的通知——一种出于爱、却也成了锁链的动作）。
终章的"分离"，本质上就是这封通知的停寄 / 你选择不再应答。他放你走，是用"不再寄通知"完成的，而非用"说教你该走"。

→ 至此"循环本身=终极逃避"和"管理员=铁驭式成全"彻底咬合：他爱你，所以一遍遍寄通知把你唤回；他也爱你，所以最后选择不再寄。

## 三、我的独立判断 / 碰撞（不照单全收）

**碰撞 1：decoy 与"自己的逃避"必须兼容，否则会滑回单元剧**
已锁纪律是"所有 decoy 的跨不过，都是玩家同一断裂的折射"。用户点出的逃避域（生活琐事/朋友/工作/陌生人/家庭）正好是折射的**分类轴**。我的提议：
- 第一幕每夜的"别人的书"= 一个逃避域的戏剧化：夜 A 信=家庭里没说出口的话；夜 B 满员书架=社交/朋友里退出的位置；夜 C 雨水=工作/学业里"熬过去但没真正面对"的一天…（具体分配待定，仅示意）。
- 玩家前期读成"别人的遗憾"，后期回照"这分明就是我每天在干的"。decoy 从"干扰项"变成"主题的镜子"。
- ⚠️ 纪律不变：仍不能每夜都把"域"写得太直白，否则提前泄底；镜子要够像别人、够不像自己。

**碰撞 2：管理员不能是"道德优胜者"，否则变说教、击穿 BE 温度**
用户说他是"很好的自己"——但"好"若写成"我留在馆里面对、你出去逃，所以我比你高尚"，就变成评判，违背 §2.3"所有人竭尽全力做到无憾"。
- 我的立场：管理员是"留下扛的那部分"，玩家是"逃开的那部分"，**两者都是同一个人、都只是累了**。他的"好"是陪伴的用心（留灯、等你），不是道德高位。终章放你走，是因为他懂你逃的苦，不是因为他比你强。
- 这条要在人设文档里写死，避免夜 C 写成"导师训话"。

**碰撞 3："行动型/实体感"必须落地为可操作的小逃避动作**
用户选 B 行动型、且厌恶纯文字（§19）。我的提议：每一夜的"正确答案"不是一句顿悟，而是**一个玩家亲手做、且以前不敢做的动作**：
- 例：夜 A 里，你终于把那封没寄出的信，在馆里**亲手投进还书口**（动作）；但终章才揭——你现实里仍没寄给活着的人。循环里"做到"是伪 HE（已锁 v2 结构），现实里"做到"才是真答案。
- 这给"行动型"实体感，也避免 v1 纯 callback 软肋（用户早否过）。具体动作清单留 M2 逐夜设计。

**碰撞 4：结尾三拍的戏份分配（17:24 已采纳为用户方案）**
用户明确：身份揭晓、逃避开心的夜、逃避真相+分离，是**三件分开的事**，中间那夜是为分离蓄力的甜。我的补充纪律：
- 别让"身份揭晓"抢了"释放"的戏——揭晓只回答"他是谁"，释放（第三拍）才回答"你走不走"。第一拍点到即收。
- "逃避开心的夜"必须写得让人羡慕/留恋，否则第三拍的"走"没有重量（呼应早前"BE要够重"的担忧：重量来自失去一个你真心喜欢的地方与人，而非惨剧）。
- 管理员全程零说教，只陪伴+寄通知；第三拍的推力来自玩家自己，不来自他的指引。

## 六、总框架 v2（待你确认 · 不落夜 A 脚本）

**A. 元引擎**
- 命题 = 逃避（机制层）。每夜 = 同一张命题考卷重发（已逃过的同类处境回声）。
- 每夜最温暖的默认路径 = 继续逃避（强引导，非惩罚）。玩家在"当下有效"里亲眼看自己一次次"倒下"、遗落版本。
- 图书馆 = 兔兔岛式温柔构造，也是"你还没走出去"的证据。

**B. 人物纪律**
- 管理员 = 你遗落的版本里"没逃、留下扛"的那个（脊柱·莫比乌斯）。
- 行为边界：只陪伴 + 寄逾期通知；零说教、零强提示、零道德高位。
- 铁驭式成全：爱你 → 一遍遍寄通知唤你回；爱你 → 最终停寄放你走。

**C. 第一幕（decoy 沉浸期，按逃避域折射）**
- 每夜"别人的书" = 一个逃避域的镜子（家庭/社交/工作/陌生人/…），前期读成别人、后期回照成自己。
- 每夜一个"行动型伪 HE"小动作（亲手做当年没做的），循环里做到、现实里没做到。
- 四区固定 + 柔性触发 + 无惩罚；节点推进制造区隔感。

**D. 第二幕（身份揭晓）**
- 夜 C：揭晓"管理员 = 自己"。只答"他是谁"，点到即收，不抢释放戏。

**E. 第三幕（终章三拍）**
- 拍 1 身份揭晓（含于夜 C / 或终章前）。
- 拍 2 逃避开心的夜：相认后特写一夜，开心地逃避，为分离蓄力。
- 拍 3 逃避真相 + 抉择分离：揭"循环即终极逃避"，门没锁、灯留原地、你独自走。

**F. BE 温度（恒定）**
- 真实·温馨·所有人竭尽全力做到无憾·但只能到这里。限度温柔，不滑向为悲而悲。
