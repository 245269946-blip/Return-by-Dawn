## 二、四维评测

### 2.1 全局叙事流畅性（夜与夜衔接、语气一致、无断点）

**评级：通过。**

- **结构节奏稳**：每一夜都严格遵循 `notice → enter → 自由探索 →（reveal）→ exit → curtain` 的节律，节奏统一，没有某一夜突然换框架或塌陷成纯文本墙。
- **跨夜语气衔接自然**：管理员的"冷淡 → 反常 → 在场 → 放手"弧光逐夜推进，不跳变。如夜 D 闸门（带你看他的来处）、夜 E 镜像（赶你走、因爱放手）、夜 F 清醒地转、夜 G/H/I 狂欢复调（"那明晚你还来还书吗？"在 G/H/I 三夜重复出现，叠加出宿命感与贪恋的重量）、终章 Z 收口——情绪曲线连贯。
- **跨夜携带真实生效**：dump 中每一夜 `[本夜线索]` 列表逐夜变长（序章 ~15 条 → 终章 ~53 条），证明 `clues/memories/hookChosenLine/revealSeen` 整套跨夜链真实并入，夜与夜"真正串联"而非各写各的。
- **无死路 / 无断点**：自由探索阶段遍历了 entry_porch / service_desk / reading_room / stacks_deep / study_zone / utility_zone / lounge_stairs / archive_lamp 共 8 区，`void_room` 全程未开启（符合铁律），每区都常驻"回到服务台"，没有任何区域把玩家困死。

### 2.2 点击互动与自由探索（能否正常推进、出口是否正确、void 是否不开启）

**评级：通过（修复 2 类真实阻塞后）。**

- **热点 / 钩子 / 近景 / 追问全链路可点**：dump 显示每夜的 `>> 热点`、钩子（`[钩子(出口型)]`）、近景（`[近景]`）、近景子热点(`· 近景子`)均被正常触发，结算页（`【结算】…`）正确弹出并 continue。
- **管理员在场模型按设计落地**：序章/夜 A/B/C 馆员恒在服务台 + 条件在 study_zone；夜 C 的 `utility_zone.with_him` 是首个"事件触发才出现"的例外（钩子结果 `moveLibrarian:"utility_zone"` 置位后才肖像可见、沉默陪伴），符合 v6 原则⑳。
- **出口机制正确**：序章/A/B/C 走服务台 `toExit` 钩子/热点；D/E/F 走 `reveal → to_exit`；G/H/I 走补回的 `leave_library`；Z 走 `prologue_book/choose_ending`。出口触发日志全部出现，`pendingExit` 门控（需配合 `settlement`）工作正常。
- **`void_room` 永不开启**：全文无 void_room 区域 dump，符合铁律。
- **⚠️ 退出时引擎 WARNING**：日志末尾有 `12 CanvasItem leaked` + `24 ObjectDB leaked`。这是 headless 测试场景（playtest_dump.tscn）卸载时的对象清理告警，**非游戏内容缺陷**，不影响游玩与评测结论；真实游戏场景 `Main.tscn` 下不触发此告警路径。

### 2.3 叙事通过性（玩家能否读懂、decoy→自我指认弧是否清晰）

**评级：通过。**

- **decoy → 自我指认的弧清晰**：前段（A/B/C）玩家读成"替别人还书"，夜 D 闸门用三张借阅卡把"借阅人全是自己"拼合出来，玩家**自己**认出"是我"——不是被告知。dump 中夜 D reveal 文本（line 791-805）层层递进：废稿划掉的手写通知（"你最近……回来看看吧"被划掉）→ 三张卡借阅人均为你、墨团"下次"是你写的 → 整间屋子的动作都是你的 → "他本就是我自己遗落的那部分"。**零提前泄漏**，完全守住 v6 自认红线（A/B/C 三夜无"其实是你"式点破句）。
- **"是我"落点扎实**：夜 D reveal 收束"我终于认出了他。不是被谁宣布，是我自己，把碎片一块块，拼回了这个人"——认知落差靠玩家主动拼合产生，而非事件炸弹，符合"认知冲击≠事件冲击"原则。
- **三物证 callback 落点准确**：终章 Z 用便签（A·关系域）/ 相册书空位（B·家庭域）/ 雨水盆边物（C·自我域）触发"正确做法"闪回，明确落在**知≠行**缝隙（"我从来都知道要怎么做，只是情绪和压力不会因为我懂了道理就消失……懂了也还是难"），**没有写成"我终于做到了"**，与 v6 红线一致。

### 2.4 叙事剧情感（认知落差是否落地、终点 BE 温度是否达标）

**评级：通过（温度达标）。**

- **认知落差落地**：夜 D 的"四天午夜通知从来不是系统群发、是他一张张写的"是全作最重的一记反转；夜 Z 的"最后一次通知没寄出——他爱你所以放手"把"想你回来"与"忍住不寄"压成同一句话，落点比直说"你该走了"更重。
- **终点 BE 温度达标**：终章收束句（dump line 1501-1503）"灯还亮着。——你终于不用等我了。……两个版本的你都竭尽全力，只能到这里。限度温柔，非致郁——不是你痊愈了，是你能往前走了。"完全命中 v6 收束句要求：**真实、温馨、所有人竭尽全力做到无憾、但只能到这里**的"限度温柔"，而非纯惨纯痛。
- **狂欢三夜的复调宿命感**：G/H/I 重复"那明晚你还来还书吗？"，在已知"是我"后仍贪恋、把续借的线拽回手里，把"贪恋被稳稳等着"的情绪写足，为终章的"放手"提供足够重量对冲。
- **双侧逃避镜像成立**：夜 I 的 `hard_book` 钩子（推到明天/今天就还）与终章 `echo_heart`（"他的声音=你逃避的回声""我不能再爱你爱到把你变成我的样子"）把玩家用旧痛逃现在、管理员用旧暖逃分离的双侧镜像同框，弧光完整。

## 三、试玩中发现并修复的真实阻塞（3 类）

### 3.1 第三幕 G/H/I 出口缺失（真实 UI 阻塞，已修）

- **现象**：三夜 JSON 有 `exit` 节点但全夜无任何 `toExit` 触发点 → 玩家在 UI 内走到收场无路可走。
- **修复**：三夜各在 `service_desk.hotspots` 追加 `leave_library`（`toExit:true` + `settlement` + 收尾叙事"那明晚你还来还书吗？"），并在 `companion` 追加对应行。复用语义，与狂欢复调一致。
- **验证**：dump 中 G/H/I 三夜均出现 `[触发出口·区域热点] service_desk/leave_library` 与对应 EXIT 文案。✅

### 3.2 系统性 `requiresFlag` vs `clue` 门控缺陷（引擎最小修复，已修）

- **现象**：引擎 `_refresh_region_controls` 的 `requiresFlag` 门控**只查顶层旗标，不查 `clues` 字典**。而跨夜携带约定里 `clues` 被携带但**不会升格为顶层旗标**——导致所有以 clue-id 作门控来源的热点（夜 C/D 的 `c_returned`、夜 B 的 `shelvedAlbum`、终章 Z 的 `c_prologue_book_waiting`）在真实游戏中**永远不可达**，终章 Z 因此**不可通关**。
- **修复**：引擎 530-532 行改为同时接受 `state["clues"].has(rf_flag)`（仅扩展门控来源，**未新增节点类型/新流程，守住"零改引擎"纪律**）；dumper 两处 `requiresFlag` 检查同步修改。
- **验证**：dump 中终章 Z 走通 `prologue_book/choose_ending → stay`，`[本夜线索]` 含 `c_prologue_book_waiting` 等全部跨夜物证。✅
- **影响面**：这是一次**系统性**修复，惠及所有 clue-id 门控热点（不限于 Z）。

### 3.3 dumper 顺序假象（误报，已绕开，非内容缺陷）

- **现象**：dumper 探索阶段按区域字典序，`service_desk` 排在 `utility_zone` 前 → `mailedLetter` 尚未置位 → `talk_librarian` 门控被跳过，造成 a/z 误报"走不到出口"。
- **修复**：dumper 新增 `_reveal_ready()`（查 `requiresClues` 对 `state["clues"]` 满足情况，替代原"拼合那一夜"按钮文案判断）+ `_re_explore_after_reveal()`（reveal 后二次探勘 `requiresReveal` 近景子热点，置位前置旗标如 `mailedLetter`），消除误报。
- **性质**：纯测试脚本逻辑问题，游戏内容本身无缺陷。

## 四、局限与建议

**局限（必须说清）**：本次是**脚本化 dumper 模拟玩家**，不是真人试玩。"剧情感"是基于逐夜 dump 出的授权叙事文本做的人工评测，衡量的是"文本在序列阅读下能否产生设计预期的落差与温度"，而非实测玩家情绪数据。要进一步验证，建议：
1. **真人 alpha 试玩**：让 3-5 个未参与开发的玩家从序章玩到终章，重点观察夜 D reveal 是否"自己拼出来"、终章是否"限度温柔而非致郁"、狂欢三夜是否"贪恋得动人"。
2. **节奏埋点**：单夜约 10 分钟的目标，建议在实机加粗略时长埋点（不强制），确认夜 A/B/C 各自不超 / 不足时太多。
3. **音频 / 白噪音层**：当前评测不含音频（框架优先于表现层，符合纪律）；待框架稳定后可补白噪音层与点击音，再单独评一次沉浸感。

**建议（可选优化，非阻塞）**：
- 夜 B 的 `shelvedAlbum` 门控此前因 3.2 缺陷不可达，修复后已可达；建议人工再读一遍该支线文案，确认其与"归家版自己"种子衔接自然。
- 退出泄漏告警（§2.2）建议在正式发行前清理测试场景卸载逻辑，避免误导。
