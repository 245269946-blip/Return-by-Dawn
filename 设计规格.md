## 1. 项目定位

- **一句话**：一款以「遗忘 — 寻找 — 继续人生」为主题的记忆探索文字向游戏；而你寻找的「自己」，早已破碎。
- **类型**：互动叙事 / 文字冒险。结构上采用 **「单元剧」手法**——每一夜是一个自洽的片段章节（一阵记忆、一封信、一个被替代的位置），但**里层真相是「所有夜都只讲同一个人（玩家自己）」**：你归还的每一本逾期书，终归是你自己遗落的一段；单元剧是容器，不是「不同人的独立故事」。**表层呈现**上，前期夜用「别人的故事 / 送错通知探索」做 misdirection（见 §2.5），先不引到自己身上。所有夜由一条**终极执念（管理员 = 你遗落的版本）**贯穿（见 §12）。
- **基调**：深夜、安静、克制、共鸣；便利店灯光般的暖与孤。入口轻、收口重。
- **核心体验**：每一夜是一次被追讨的记忆。玩家在图书馆里解谜、探索、重建，并在天亮前做出属于自己的人生选择。
- **北极星（To the Moon 向）**：核心不是机制，是「讲好一个故事」。机制（解谜、时钟、重建）皆为叙事服务；每一次夜都通向一个结果，而非「通关 / 失败」。
- **平台（建议）**：本地网页原型（HTML + CSS + JS，无后端，双击即玩），便于快速验证氛围与迭代。

## 2. 世界观与核心隐喻

- **图书馆**：收藏「被遗忘的自己」的切片。你遗落的部分，被它默默记录。
- **书名是自己（被追讨的是你遗忘的一部分）**：那本「逾期未还的书」严格说**不是你，只是你遗落、遗忘的一段**——一封没送出的信、一个没说出口的人、一段你不敢翻的自己。你不断来还的，是这些碎片，不是「你自己」整体。
- **逾期通知**：午夜送达，天亮前须至馆归还。
- **归档**：想通了一些事，记忆被收存。它**不是惩罚**，而是和解 / 放下的一种可能。
- **不愿归还**：书里是你舍不得交出的记忆——这正是冲突的来源。
- **终极执念（本季主线，后期揭晓，见 §12）**：图书管理员 TA 是**你遗落最久的那个版本**——过去的、或未来的你，也是「逾期最久的那本书」。TA 的任务是陪你走完每个夜、悄悄把你引回自己，却**绝不能让你发现**引路者就是你自己。
- **元层伏笔（第二季）**：那个邮箱为何总在？命运的问题会重复抛出，直到你给出正确答案。本季只留缝隙。

### 2.1 主角底色（本作情感真相 · 已定）

> **主角「自己早已烂透 / 默哀心死了」。**

这是全书的情感地基，也是 BE 的真实内核：

- 玩家表面在「寻找自己、和解、继续人生」，但底层真相是——被追讨的那个自我核心，早已破碎、心死。你不是在找回一个可以痊愈的自己，而是在**终于看见那个已经烂掉的、回不去的自己**。
- 这正是 To the Moon 的写法：记忆机给了 Johnny 一个圆满的谎（他想去月球、和弟弟重逢），但真实的落点永远是「有些遗憾就是回不去」。本作里，主角一直在用**投射出来的暖（见 §10）**自我安慰——以为和解了、以为还书了、以为那人还在——但那些温暖大多是自己给自己的安慰剂。
- **关键纪律**：这个「自我早已破碎」的真相**不是前期主线**，前期只立住「馆员」人设、走「面对逾期内容、一本本还书」的核心循环；真相要到夜 C / 终章才以玩家自行拼合的方式揭晓。前夜只在缝隙藏微小异常。
- **母亲章 / 暖心和解章的摆位（重要澄清）**：母亲母子那段「与吵架的妈妈和解、说出『我今晚回家吃饭』」只是**「投射式小结局」装置的一个旗舰示范**（见 §10），是让玩家去探索的「距离」，**不是脊柱**。全书脊柱始终是管理员 = 你遗落的自己。

### 2.2 莫比乌斯环（架构核心，精确版）

- 图书馆收藏「被遗忘的自己」的切片（见 §2）。但请精确：**书不是你，只是你遗忘的一部分**——一封没送出的信、一个没说出口的人、一段你不敢翻的自己。你每夜来还的，是这些**碎片**。
- 守着所有碎片、等你把它们一本本送回的，是被遗落最久的那部分：**管理员 TA 是你自己的一个版本**（过去没走成的那条路，或未来若想通了会成为的样子）。TA 留在馆里，因为「你」还没回来。
- TA 的任务：陪你走过每个夜，把碎片递到你手边，让你一点点重建、靠近真实的自己——**但不能让你发现引路者就是你自己**。一旦提前发现，环就断，你也失去「自己找回自己」的可能。所以 TA 永远扮演「馆员」，永远说「灯还亮着」，从不说破。
- 没有「里」与「外」的硬边界：你归还的碎片来自你，守碎片的人也是你的一部分。环只有一面——只是「你」以不同形态分布在环上：**碎片是你遗忘的，馆员是你留下的。**

### 2.3 情感温度原则（BE 的真实落点 · 已定）

本作 BE 的落点，**不是「纯惨纯痛」，而是「真实、温馨、所有人竭尽全力做到无憾、但只能到这里，只能到这里了」**——这是用户对结局的最高指令，覆盖 §2.1 的「自我早已破碎」与 §12.2 的「To the Moon 式 BE」：任何一处的痛都必须落在这句里，不能滑向为悲而悲。

- **真实**：不粉饰遗憾，也不放大疼痛。痛是「回不去」，不是「被虐」。
- **温馨**：全程有 TA 的陪伴、有那些微小的暖（「灯还亮着」、一句没说出口但终于写下的话）。暖**是真的**，不是 §10 那种要被揭穿的安慰剂假暖。
- **所有人竭尽全力做到无憾**：每个角色、每段记忆，都已经在自己能做的范围内做到了最好——书还了、信写了、该说的话在心里说过了。没有「如果当初再努力一点就能圆满」的 if，因为**他们已经尽力了**。
- **但只能到这里，只能到这里了**：BE 的根是「限度的温柔接受」，不是「失败的打击」。不是「你不够好所以输了」，而是「你用尽了全部，故事也只到这一页」。读者 / 玩家的酸涩来自「他们明明都尽力了，可还是只能停在这里」，而不是来自残忍。

> 与 §2.1 的关系：§2.1 的「自我早已破碎」是真相的地基，但地基之上要长出的是这句「我们尽力了，也只能到这里」的温柔 BE，而非致郁宣言。写 §2.1 时务必守住这条温度线（见 §16）。

### 2.4 故事是「一个人的故事」（结构澄清 · v0.7 已定）

- **核心确认**：本作从头到尾是**一个人的故事**——玩家自己的。每夜的「逾期书」不是陌生人的遭遇，而是**玩家遗落的人生碎片**（一封信、一个没说出口的人、一段不敢翻的自己，见 §2.2）。
- 因此「单元剧」只是**叙事容器**：用一章一章自洽的片段，把同一个人的不同伤口依次摊开；它**不是** anthology 意义上「每夜换一个主角 / 讲别人的故事」。所有夜共享同一个「我」，最终收束到同一个真相（管理员 = 你遗落的版本，§12.2）。
- 这一条同时回收了 §1 的类型表述与 §12.2 的终极执念：书 = 遗忘的碎片（你的一段），馆员 = 留下的自己（你的一个版本），环上只有「你」这一个人。
- **表层 / 里层的分层（v0.8 关键澄清）**：「一个人的故事」是**里层终极真相**（脊柱不变，见 §12.2 莫比乌斯环）。但**表层呈现**上，早期夜要给玩家一种「我在帮别人 / 在解一个送错通知引发的谜」的错觉——你看到的像是陌生读者的故事、像是系统错投的逾期单。这是 To the Moon 式的 misdirection：**前期越像「别人的事」，二三幕回收成「其实是你自己」时的翻转越沉**。所以早期每夜**先不引到自己身上**，保持「迷雾但轻松前进」；真正的自我指认留到夜 C / 终章。详见 §2.5。

**图书馆 = 管理员的「兔兔岛」（温柔的构造 · v0.7 已定）**

- 对标《去你的岛》里**王结香为殷显编织的兔兔岛谎言**：那座岛是 Jiexiang 给 Yin Xian 的一个温柔构造，让他有地方可回、有念想可抓，才撑得过丧亲之痛。本作里，**整座图书馆、那盏永远为你留的灯、以及「逾期书可以一本本还回去」这件事本身，就是管理员（你留下来的那一半）为你编织的温柔构造**。
- 它的功能与兔兔岛同构：**给那个破碎的、走不出去的自己，一个可以反复回来的地方**，直到你终于愿意往前走。
- 因此管理员**没有欺骗的恶意**，只有陪伴的用心——TA 不是「藏着真相害你」的人，是「替你留着灯、等你准备好」的人。这一定位让 §2.3 的「温馨 / 陪伴」有了具体落点，也把夜 C 的揭晓从「被背叛感」扭转为「被陪伴感」（见 §12.3）。
- 收束时，TA 会像 Jiexiang 最终不得不让 Yin Xian 面对现实那样，轻轻放下这个构造，说出那句「我可以一直陪伴在你身边，但你得自己往前走了」（见 §12.2 收束句）。

### 2.5 叙事节奏与装置：表层 decoy、思维定式、伏笔回收（v0.8）

参考 To the Moon 的叙事节奏优化——前期轻、中期累积、后期落刀。据此把「一个人的故事」拆成**表层 misdirection + 里层真相**两层，并加三类装置：

**1) 表层 decoy：早期夜 = 「别人的故事」/「送错通知的探索」**
- 第一幕的每一夜，玩家进入图书馆看到的**先是「别人的故事」**：某位陌生读者的逾期书、某段与你无关（看似）的遗憾；或由一张**送错的逾期通知**引发——通知本不该来你手上，你出于好奇 / 责任感去馆里探个究竟。
- 这些夜**都是能解决的伪HE（阶段性好结局）**：谜团解开、书还了、当事人（看似）释然。玩家此时**不引到自己身上**，体验是「迷雾但轻松前进」的治愈探索感。

**2) 思维定式装置（为翻转蓄力 · 关键 · v0.9 校正）**
- **定式固定在「流程 / 事件」上，不固定在「具体情节」上。** 每一夜都复现同一组**事件拍子**——① 收到逾期通知 → ② 前往图书馆 → ③ 自由探索 / 解谜 → ④ 还书（问题=谜题闸门）→ ⑤ 知道被遗忘的事 → ⑥ 三选一处置。玩家对「这五拍会来、会循环」形成预期，这就是思维定式。
- 但**每一拍的「样子」必须每夜不同**：通知里的书名、你「接到通知」的方式、图书馆当夜的前台（下雨白噪音 / 午后管理员打盹 / 后期管理员不在了你得自己找）、探索里的谜题、结尾三选一的语境——全部由当夜内容决定，不可复用同一段具体情节。
- 翻转不在「某句暖词第一次被改写」，而在「**拍子照旧，但某一夜某一拍的质感第一次塌了**」：开头照旧、结尾却第一次「还不了」/「灯灭了一瞬」/ 三选一第一次给出 BE 重量。因为玩家早已信任这五拍，信任被打破时才刺到人（见 §4.1）。

**3) 伏笔回收：碎片阶段的 BE 种子**
- 早期「别人的故事」里，要**悄悄埋下属于你自己的 BE 伏笔**：一个反复出现的名字 / 笔迹 / 物件特征、管理员一句看似对旁人说的旁白其实也适用于你、一张送错的通知上印着你自己的 ID。前期读来像氛围，二周目 / 夜 C 重读才是线索。
- 二三幕：这些伏笔逐一回照、收束到「其实是你」；同时整体氛围从轻松转向 BE 预兆——不那么直说，但光开始不稳、馆员的话开始有重量、某本「别人的书」越来越像你的。

**4) BE 温度校准（两部电影作为 north star · v0.8）**
- 《超凡蜘蛛侠2》：BE 来自「**我救不了所有人**」的负罪与无力——不是坏人作恶，是自己竭尽全力仍留不住该留的人。本作夜 C / 终章的痛应同构：你尽力了，可那本最想还的书、那个最想见的人，还是回不来。
- 《海边的曼彻斯特》：BE 来自「**伤痕无法被治愈，只能到此为止**」——主角最终放弃争夺监护权，不是不爱，而是清楚自己「beat it」不了。这正是 §2.3「只能到这里」的范本：没有强行治愈，没有「只要努力就能好」，只有带着碎痕继续活。本作终章落点应对标这种**限度的温柔接受**，而非致郁打击。

## 3. 设计支柱（不可妥协）

1. **讲故事优先（北极星）**：互动服务于叙事推进与选择，不喧宾夺主。
2. **叙事性解谜推动剧情**：保留 puzzle，但谜题是「情绪的钥匙」——解开即推进或揭示一段记忆，不为炫技。
3. **无惩罚式失败（实验哲学）**：每一次夜都有结果。**「每一次实验都是有结果的，无论产量高低、数据好坏、成率高低，都是在前进。」**——故本作没有「失败」结局，只有不同的「实验结果」（归档 / 归还 / 私藏 / 隐秘）。
4. **每夜独立 + 一条终极执念贯穿**：单元剧结构保证每夜新鲜，但所有夜围绕同一终极执念（见 §12），最终收束为一句话。
5. **留白即节奏**：沉默、空场、暖光偶灭，是体验的一部分，不是「没内容」。
6. **氛围即玩法**：深夜图书馆的克制安静本身，就是体验。
7. **假结局是结构的一部分（投射式装置）**：每章可先给一个温暖的假结局（见 §10），后期用缝隙回照揭穿为投射 / 误认，制造 To the Moon 式「真实 / 暖心 / 遗憾 / 意难平」的回照。

## 4. 核心循环（单夜）

```
收到通知 ──▶ 动机引入 ──▶ 探索 / 解谜 ──▶ 重建（揭示：常先给暖假结局） ──▶ 天亮前抉择 ──▶ 结果（结局，后期回照揭穿）
```

| 阶段 | 作用 | 说明 |
|---|---|---|
| 收到通知 | 入口钩子 | 午夜催还单，建立仪式感 |
| 动机引入 | 差异化起点 | 这一夜「为什么不同」：正常还书 / 无人接待 / 位置被替代…… |
| 探索 / 解谜 | 空间互动 + 叙事性谜题 | 在馆内移动、阅读、拾取记忆碎片；谜题解开即推进剧情，非障碍 |
| 重建（揭示） | 叙事拼合 | 集齐碎片 → 触发一段完整回忆；**常先呈现一个温暖的假结局**（见 §10），真相关锤留到后期回照 |
| 天亮前抉择 | 玩家代理 | 还 / 悄悄带走 / 销毁通知 —— 人生选择 |
| 结果（结局） | 收束 | 四种结果之一，皆有效；并埋下元层 / 终极执念种子 |

**两夜示例（证明每夜可不同）：**
- **夜 A《夹在书里的信》**（重建向）：正常来还书，最后一刻翻开书发现妈妈写的信，逐步重建与妈妈的点滴。其「和解」可先给暖假结局，后期揭穿为投射（见 §10 旗舰范例）。
- **夜 B《满员的书架》**（探索冒险向）：无人接待，书架上你的位置已被替代，踏上寻找个人「位置」的旅程。

### 4.1 单夜流程模板（v0.9 · 夜 A demo 已按此落地）

每一夜都遵循同一套**有逻辑、非纯呈现**的流程，且**流程固定、情节每夜不同**（见 §2.5②）。夜 A demo 已实现全套：

1. **固定事件 · 收到逾期通知（每夜复现，吸睛钩子）**：「午夜收到催还单」这个**事件**每夜都来，但**书名、你「接到」的方式每夜不同**（门缝滑入 / App 弹窗 / 纸鹤……）。是仪式钩子，也是定式的锚；玩家对「又要收通知了」形成预期。
2. **固定事件 · 前往图书馆（每夜复现，前台不同）**：「推门进馆」这个**事件**固定，但**当夜前台每次不一样**——雨夜白噪音的安静 / 午后管理员眯着打盹 / 后期管理员不在了你得自己摸黑找。建立安全感，也用前台差异暗藏情绪位移。
3. **分场景自由探索 + 叙事性解谜（自由度高，锈湖 / 锈锚式房屋解密叙事）**：馆内多个场景（阅览区 / 旧书架 / 归还箱 / 靠窗座……），玩家**任意顺序走动、可重访**；每处是一个**轻量解密**，解开给一条**阶段线索**（如书脊拆字排成收信人、旧书按年份排露出夹层日期、信纸碎片拼出说话人）。解谜是「情绪的钥匙」，不是障碍（见 §3.2）。
4. **还书流程的问题 = ③里的谜题（闸门）**：走向还书台时卡住，而卡住的**就是探索里某道谜题的收口**——例如归还箱要输入「应还日期」，日期得从旧书架夹层那道谜得来；线索没找齐就还不了。问题即情节，逼出探索与抉择。
5. **写着玩家名字 / 特征的书（降为「某个故事的收束点」，非全季固定装置）**：它可以是某一夜的关键道具（如夜 A《__NAME__的夏天》夹着未寄出的信），但**不强求每夜出现、不充当贯穿铰链**——它可以只是某一夜的结局落点之一。
6. **结尾：知道书该回的地方 + 想起被遗忘的事 → 三选一处置（建立定式的「拍子」，不靠固定暖句）**：玩家终于**知道书该归还的处所，也想起被遗忘的那件事**，此时才做抉择——**归还 / 偷偷带走 / 销毁**。三选一这「一拍」每夜都有（定式在拍子），但语境与重量每夜不同；早期是伪HE阶段好结局，二三幕才第一次承重。

## 5. 架构：引擎与内容分离

**原则：一次开发通用引擎，无限扩展内容模块。**

- **通用引擎（一次开发）**：空间导航、碎片收集、叙事性谜题判定、选择系统、重建判定、时钟 / 行动预算、结局解析、元层存档。
- **内容模块（可无限扩展）**：每个「夜 / 本」= 一个 JSON 数据文件，零代码改动即可新增故事。
- **收益**：夜 A、夜 B 只是两个 JSON；加新故事不必重写引擎。

## 6. 单夜数据结构（详）

一夜 = 一个 JSON 模块，建议字段如下（示例以夜 A 为骨架）：

```json
{
  "id": "night_mother_letter",
  "title": "夹在书里的信",
  "theme": "母爱 / 遗憾 / 和解（暖假结局示范）",
  "anchor": {
    "object": "夹在书里的信",
    "line": "我本来想亲口告诉你……",
    "reveal": "结局揭晓：这封信，是写给一个已经听不到的人的"
  },
  "notice": {
    "time": "00:00",
    "text": "午夜逾期通知：您有一本名为《自己》的书逾期未还。天亮前请至图书馆归还，逾期将被归档。"
  },
  "hook": "这一夜你照常收到通知，却觉得书比往常更沉……",
  "space": {
    "start": "entrance",
    "nodes": {
      "entrance": {
        "name": "馆前",
        "desc": "……",
        "exits": { "推门进馆": "reading_room" },
        "actions": []
      },
      "reading_room": {

        "desc": "……",
        "exits": { "走向书架": "shelf_area", "到还书台": "return_desk" },

        "actions": [
          { "id": "read_letter", "label": "翻开书", "requires": "f1", "unlocks": "memory_1" }
        ]
      },
      "shelf_area": { "name": "书架区", "desc": "……", "exits": {}, "fragments": ["f2"] },
      "return_desk": { "name": "还书台", "desc": "……", "exits": {}, "decision": "final_choice" }
    }
  },
  "fragments": {
    "f1": { "text": "书页间露出一角泛黄的信纸……", "unlocks": "memory_1" },
    "f2": { "text": "……", "unlocks": "memory_2" }
  },
  "memories": {
    "memory_1": { "title": "妈妈的字迹", "text": "完整回忆文字……（先给暖假结局）", "points": 1 },

  },
  "choices": {
    "final_choice": {
      "prompt": "天快亮了。你打算怎么处理这本书？",
      "options": [
        { "id": "return", "label": "放回还书台", "tags": ["return"] },
        { "id": "take",   "label": "悄悄带走",   "tags": ["take"] },
        { "id": "destroy", "label": "撕毁逾期通知", "tags": ["destroy_notice"] }
      ]
    }
  },
  "endings": {
    "archived": { "cond": "reconstruct < 2 && dawn", "title": "归档", "text": "……" },
    "released": { "cond": "reconstruct >= 2 && choice == 'return'", "title": "归还", "text": "……" },
    "kept":     { "cond": "reconstruct >= 2 && choice == 'take'", "title": "私藏", "text": "……" },

```

**字段说明**
- `anchor`（新增）：每夜的**执念锚点**，见 §12。
- `space.nodes`：空间图。每节点含 `desc`（环境描写）、`exits`（出口→目标节点）、`fragments`（可拾碎片）、`actions`（条件互动，含叙事性谜题）、`decision`（指向最终抉择）。
- `fragments` / `memories`：碎片拾取后解锁对应 `memory`；`memory.points` 计入重建度。
- `choices`：关键抉择点（可多个，夜 B 的探索中也可插入支线抉择）。
- `endings.cond`：引擎可求值的伪表达式（见 §9）。

## 7. 「重建」机制（叙事性解谜）

- 玩家在探索中获得**碎片**（文字片段 / 线索），进入「记忆拼图」栏。
- 集齐某段记忆所需碎片 → 触发一段**完整回忆**（如夜 A 拼出和妈妈的点滴）。
- **叙事性谜题**：部分碎片 / 回忆需通过「情绪的钥匙」解开——例如某句旁白提示、某个物件组合、某段记忆的前置条件。谜题不考智商，考「你愿不愿意往那个方向想」。
- **重建度** = 已解锁 `memory.points` 之和。
- 重建度 + 最终抉择 → 查 `endings` 表出结果。
- 设计意图：重建不是收集癖，而是「你愿不愿意正视这段记忆」的隐喻。无论重建度高低，都是这夜的一个有效结果。

## 8. 时钟与行动预算（无惩罚）

- **时间轴（建议）**：00:00 收到通知 → 05:30 天亮。
- **推进方式**：每个「有效行动」（移动 / 阅读 / 拾取 / 解谜 / 抉择）消耗一定剧情时间（如 +10~20 分钟），界面显示「距天亮还有 X」。
- **不绑真实秒数**：不是动作游戏的实时倒计时，而是**行动预算**——你能在天亮前探索的次数有限。
- **结算点，非惩罚**：天亮（预算耗尽或主动走向还书台）= 进入结局判定。预算不足只意味着你走向一个**不同的结果**（如「归档」），而不是「游戏失败」。正如实验哲学：产量低也是数据。
- **留白空间**：允许玩家在某个节点「什么都不做」——沉默本身可触发管理员的旁白或一段空场，是节奏的一部分。

## 9. 多结局矩阵（皆为结果）

以夜 A 为例，结局由 **重建度 × 最终抉择** 决定：

| 重建度 ＼ 抉择 | 归还 | 带走 | 销毁通知 |
|---|---|---|---|
| 充足（≥2） | 归还 | 私藏 | 隐秘（若已读信） |
| 不足（<2） | 归档 | 归档 | 归档 / 隐秘边界 |

> 四种结局**皆为有效结果**，无优劣、无「失败」。归档不是惩罚，是「这夜你还没准备好，但仍在前进」。

- `cond` 由引擎求值，支持 `reconstruct` / `choice` / `has('memory_x')` / `dawn` 等变量与逻辑运算。
- **元层种子**：特定隐秘结局中，露出管理员「无借出记录 / 想不起自己名字」等微小异常，为夜 C / 终章的 §12 揭晓埋缝；邮箱留作第二季。
- **终极执念种子**：每个结局可携带一句与 §12 终极执念呼应的回响，让单元剧在情绪上收束到同一点。
- **投射式回照钩子**：若本章用了 §10 暖假结局，可在某一结局里预埋「缝隙」——一句看似温暖的台词，在二周目重读时变成揭穿的种子（如「你今晚回家吃饭」其实是路边陌生母子的对话）。

## 10. 投射式小结局（可复用叙事装置）

> 本装置是 v0.6 的核心新增。它把上一轮讨论里「母亲母子和解」的例子，从「一章剧情」提炼为**可复用的结构装置**：每章先给玩家一段温暖的、看似圆满的假结局，后期用缝隙回照拆穿它是投射 / 误认。

### 10.1 装置定义

- **先给暖假结局**：每一夜（或某一类夜）在重建揭示时，先呈现一个让玩家以为「这一夜完成了、和解了、放下了」的温暖瞬间。例如：与吵架的妈妈和解，主角终于愿意开口说出那句「我今晚回家吃饭」。
- **后期揭穿（缝隙回照）**：在同章的缝隙、或后续章节的回照里，用具体、可被玩家自行拼出的细节拆穿——那幕温暖是**投射 / 误认**：
  - 后面情节里发现，家里摆着的其实是妈妈的**墓碑 / 灵牌**；
  - 终章揭穿：第一章那场和解**从未发生**，主角只是把**路边看到的陌生母子的经历，当成了自己的**；
  - 那句「我今晚回家吃饭」，从来就没有真正说出口过。
- **目的**：制造 To the Moon 式「真实 / 暖心 / 遗憾 / 意难平」的回照——让玩家先尝到「治愈」的甜，再用真实反咬一口，体会「你以为放下了，其实只是自己骗自己」。

### 10.2 旗舰范例（母亲章的摆位）

母亲章是「投射式小结局」的**旗舰示范**，不是全书脊柱：

- 第一幕（暖假结局）：小结局，主角与妈妈和解，说出「我今晚回家吃饭」。
- 缝隙 / 后期揭穿：后续发现家中墓碑 / 灵牌；终章确认那场和解是路边陌生母子的投射，从未属于主角。
- 它用来证明装置可行、可动人；但全书贯穿的仍是管理员 = 你遗落的自己（§12）。

### 10.3 可复用性与纪律

- 这是**装置，不是某一章专属**。信、人、位置……每一章的「逾期内容」都可用暖假结局开场、后期揭穿。
- **但需控频（见 §16 风险）**：若每章都做投射翻转，会疲劳、变廉价。旗舰只用母亲章等 1–2 处，其余章以「轻度暖假结局 + 留白」为主，避免每章都抖包袱。
- **玩家代理**：揭穿必须由玩家在记忆里自行拼合，不靠旁白直说「其实那是假的」。
- **与 §12 终极执念的关系**：微观的暖假结局，呼应宏观的「自我早已破碎」真相——你投射出的每一段温暖，都是对那个烂掉的核心的代偿。装置的终点是：玩家认出「我一直在用假的暖，喂真的空」。

## 11. 视觉与 UX 方向

- **色彩**：近黑深底 + 一束暖光（便利店灯光感）；文字米白 / 暖灰，强调色极克制（如旧纸黄）。
- **字体**：叙事用衬线（宋体 / Noto Serif）；系统通知用等宽（催还单即视感）。
- **布局**：单列居中，留白多；右侧或底部常驻「钟」与「重建进度」。
- **动效**：极轻。翻页、灯光微呼吸、进度条缓动。无炫技转场。
- **声音（待定）**：极简钢琴 / 环境底噪是 To the Moon 感的重要来源；**沉默是节奏的一部分**——允许长段无音乐，只在情绪高点落一个音。建议 M2 评估是否引入。
- **组件清单**：逾期通知卡、空间描述区、出口 / 动作按钮、记忆拼图栏、钟、结局页、图书管理员旁白条。
- **明确不做（anti-AI / anti-电商）**：无渐变轰炸、无 emoji 堆砌、无 CTA 按钮、无「立即体验」式营销语、无日式符号堆砌。

### 11.1 行文语气（参考《去你的岛》）

行文感觉参考番大王《去你的岛》——一部以奇幻 / 成人童话般轻盈开篇、层层揭开真相、于平淡现实中落刀、欧亨利式收束的作品（微苦带甜、克制不浮夸、疏离又温柔）。落到本作：

> **参考的边界（已澄清）**：借的是《去你的岛》的**整体故事设计 + 叙事形式**（轻盈入口 → 层层揭开 → 平淡现实落刀 → 欧亨利式收束），**不是**逐字模仿它的行文词句。行文可以学它的「神」（克制、疏离又温柔、微苦带甜），但落点须按本作自己的情感温度走——见 §2.3。

- **入口轻、收口重**：像《去你的岛》以荒诞轻盈的入口（床破窗飞天、前男友变成兔子）起笔，再一层层揭开悲伤真相；本作用「午夜逾期通知」这个日常又微微超现实的钩子开场，前几夜轻，后期才落刀。
- **克制不浮夸**：疏离又温柔；允许一点荒诞与日常并存（图书馆、逾期通知、被替代的书架位置），但情绪内核必须是真实、生活化的。不煽情、不为虐而虐。
- **微苦带甜的余味**：暖光与孤并存；即使揭穿是投射、即使自我早已破碎，也要在痛里留一点「灯还亮着」的暖。
- **欧亨利式收束**：单章可在「似乎和解」处反转，全季亦可——在玩家以为治愈的那一刻轻轻反咬，留下微微酸涩又微微释怀的余味。
- **忌**：长篇独白式心理分析、过度解释、把「TA 是你」当抖包袱秀作者聪明。

## 12. 执念锚点（每夜）与终极执念（贯穿）

### 12.1 每夜执念锚点（必填字段 `anchor`）

每一夜需要**一个具体的执念锚点**，由两部分组成：
- **一个实物**（`object`）：夜里有形可触的东西，承载情绪。
- **一句反复回响的话**（`line`）：随夜推进，这句话在结尾获得新意义（`reveal`）。

锚点不是线索，是**该让人鼻酸的东西**。它是夜 A 的信、夜 B 的那个「被替代的位置」……

**夜 A 锚点（方向已定，终稿待你定）：**
- `object` = 夹在书里的信（不变）
- `reveal` = 结局揭晓：这封信，是写给一个已经听不到的人的（不变；且与 §12.2 终极执念形成「微观→宏观」的呼应）

**回响句 `line` 的方向（你来做终稿，我给方向 + 候选）：**
- **方向**：这句要在两层都成立——第一遍读，像在说书里那位 owner（妈妈）没说出口的话；通关一遍、知道 §12.2 后重读，它要悄悄变成在说管理员（那个你遗落最久、留下来等你的自己）。它该围绕「没说出口 / 没还回去 / 没翻到的那一页」打转。
- **候选 1**：「逾期的不只是书。是你一直没还给谁的，那句话。」
- **候选 2**：「有些话夹在书里，不是忘了说，是不敢翻到那一页。」
- **候选 3**：「他写这封信的时候，以为还有改天。」
> 挑一句，或告诉我往哪个方向推，你来收尾。

### 12.2 终极执念（贯穿全季 · 已定主线，但属后期揭晓）

**已选定：候选 C 升级版 ——「管理员 = 逾期最久的那本书 = 玩家自己遗落最久的版本」，且这一点是后期才揭晓的真相。」**

> **⚠️ 结构顺序（本季最重要的一条纪律）**：这个「引路者就是你自己的一部分」的真相，**不是前期主线**。前期夜 A–B 只做两件事——（1）立住「图书管理员」这个**人**：安静、不说教、偶尔戳破，是陪你走完每夜的馆员；（2）走通核心循环：一封封面对「逾期内容」、一本本还书，让玩家先爱上这个节奏与人。直到夜 C / 终章，才让玩家**自己拼出**那个环；前夜只在缝隙里藏微小异常，绝不提前说破。

**为何往 To the Moon 走，且落点是 BE：**
它把「陪伴者」本身变成谜底，且谜底是你自己的一部分。你以为在帮别人还书，最后发现那个一直陪着你、从不说破的人，也是你。但本作的结局不是「找回自己、阖家团圆」式的圆满，而是 To the Moon 式的 **真实 / 暖心 / 遗憾 / 意难平 BE**：

- **真实**：揭示不靠神转折，而是你早就在碎片里见过的东西（那封信、那个人、那件没做完的事，以及 §2.1 那个「自己早已烂透 / 默哀心死」的底色）。
- **暖心**：全程是 TA 的陪伴——那个留下来等你的你，从未离开过。
- **遗憾 / 意难平**：你终究无法「真正归还」那个被遗忘的自己，也无法让逝去的人、错过的事重来；更刺痛的是，你一直以为在「寻找 / 和解」的那个自己，其实**早已破碎、心死**（§2.1）。你能做的，只是**终于看见、终于承认，带着 TA 给的那点暖继续走**。这正是 To the Moon 的核——记忆机给了 Johnny 一个圆满的谎，但真正的落点永远是「有些遗憾就是回不去」；本作把这句推进一层：你连「值得被治愈的自己」都未必拾得回来，但你仍能走。

**「不能让玩家发现」是设计约束，不是剧透禁忌：**
1. **先依恋，后揭晓**：夜 A–B 管理员完全是「人」、是馆员，节奏里无「TA 是你」的明示。揭晓只在夜 C / 终章，以**玩家自行拼合**的方式呈现（线索回照、镜像选择），而非一段「我其实就是你」的独白。
2. **前夜只埋微小异常**：TA 没有借出记录 / 想不起自己名字 / 目录里一条没有应还日期的条目 / 你某次抉择后 TA 的反应与你记忆里的某次重合——初读像「氛围」，重读、二周目才是线索。
3. **终章落 BE，但「如何放手」的代理权交还玩家**：BE 是**设计方向**（真实 · 暖心 · 遗憾 · 意难平的结局），但玩家在终章仍要选择**怎么对待这个环**——「看见并放手」（带着暖继续走）/「再陪 TA 待一夜」（多给自己一点时间）。两种走法都通向 BE，差别只在你与这段遗憾和解的姿态；作者不替你决定姿态，但结局「意难平」的基调已定。（注：单元夜的四种结果仍皆为有效结果，见 §9；此处说的是**全季终极弧光**的落点。）
4. **夜 A 的信是种子**：一封没说出口的信（微观）→ 一个你没还的人 → 收束为「你没还的，是你遗忘的自己的一部分」（宏观）。「逾期」= 我们因为不敢面对而一直留着的东西：一封信、一个人、一段自己。

**BE 预兆与伏笔回收（v0.8）**：早期 decoy 夜里埋的「别人的故事」伏笔，在夜 C / 终章逐一回照成「其实是你」；BE 温度校准见 §2.5④，以《超凡蜘蛛侠2》的「救不了所有人」负罪感与《海边的曼彻斯特》的「伤痕无法治愈、只能到此为止」为 north star，确保落点落在 §2.3 而非致郁。

### 12.3 管理员之夜（夜 C · 揭晓身份的一环，但非结局）

> 这一段是本轮澄清的核心：管理员身份**不是故事的结局**，只是「重新认识自己」的一环，是继续让人走下去的其中一个篇章。

- **那本被遗忘的书**：夜 C 的逾期书，是揭晓管理员身份的钥匙。翻阅它，玩家发现管理员竟与**自己息息相关**——发起「这个熟悉的陌生人，究竟是谁」的疑问。
- **可塑造成「人贩子」假象（red herring）**：前期可刻意布置紧张线索，让玩家一度误以为管理员是拐走 / 夺走自己某段人生的人（人贩子意象），制造误认与悬念；待身份揭晓，这个假象反转为「他其实是留下来守着你的那部分自己」——误认本身就是投射装置的变体。
- **身份揭晓 ≠ 结局**：认出管理员 = 你遗落的版本之后，故事**不收尾**。它只是让你「重新认识自己」的一格台阶：你带着这个认知，选择要不要再陪自己走一夜、要不要继续往前。BE 的最终落点仍在 §12.2——承认自我早已破碎，然后带着暖继续走。
- **纪律**：人贩子假象是氛围 / 悬念手段，不是类型转向；揭穿后必须落回「自我指认」与「继续走」，不可滑向悬疑惊悚。
- **BE 预兆（v0.8）**：夜 C 起，前期 decoy 夜埋的伏笔开始回照；整体从「轻松前进」转向「BE 预兆」——光开始不稳、口癖渐重、某本「别人的书」越来越像你的。落刀但守住 §2.3 陪伴感。

**收束句（全季回响 · v0.7 修订）**：终章的收束须聚焦——**「我可以一直陪伴在你身边，但你得自己往前走了。」** 对标《去你的岛》王结香对殷显说出口的那层意思：Jiexiang 用兔兔岛这个温柔构造陪 Yin Xian 走过了最痛的一段，但终归要让他自己迈步。本作里，管理员（你留下来的那一半）以同样的方式，在终章轻轻放下「馆 / 灯」这个构造，把往前走的代理权交还你。

- 与 §13 口癖「灯还亮着」的咬合：终章收束建议写成两截——
  > **「灯还亮着。——我可以一直陪伴在你身边，但你得自己往前走了。」**
  前半截是贯穿全季的口癖收束为「灯仍在」；后半截是新立的核心台词，把「陪伴」与「你必须自己走」并置，正是兔兔岛谎言收束时的那一下。
- 它应是**暖的、遗憾的、不圆满的，但绝不致郁**：在 §2.1「自我早已破碎」的真相下，它是一句面对破碎自我时的**陪伴 + 放行**，不是治愈宣言，也不是抛弃。其情绪温度须严格落在 §2.3 的「真实 · 温馨 · 所有人竭尽全力仍只能到这里」上——酸涩来自「TA 明明陪到了最后，可你还是只能独自往前」，不是来自残忍。

### 12.4 标志物（motif · 待定，不强求，后置 · v0.7）

- 类比《去你的岛》的**兔子**与**结香花**（贯穿全作的实物意象），本作也可有一个反复出现的标志物，作为情绪的物化锚点。
- **当前定位（用户明确）**：这个不强求，**现在定也早了**。等夜 A–B 的文字跑出来、自然浮现某个反复出现的小物（或某个动作 / 气味 / 光），再提炼为标志物，比硬塞更稳。
- **天然候选（不急于拍板）**：①「灯」——本就已作为口癖与核心意象存在，最顺；②「逾期通知卡」——每夜出现的实物，可承载「被追讨的自己」；③ 某件管理员总在摩挲的小物（待夜 C 人设定稿后浮现）。

## 13. 图书管理员：陪你走完这夜的人

类比 To the Moon 里 Neil 与 Eva 两位医生——一边陪伴、一边陪你挖记忆。图书管理员承担这一角色，职能分三层：

1. **旁白者**：描写空间状态、给出环境氛围；在沉默处落一句，不在热闹处抢戏。
2. **镜子**：把你刚才的选择平静地映回给你（如「你把它放回去了。和上次一样。」），不评判，但让你看见自己。
3. **渐进揭示（也是「不能让你发现」的表演）**：管理员 TA 自己就是那本逾期最久的书，且是玩家自己遗落最久的版本（见 §12.2）。前夜只以「普通馆员」身份出现、立住人设、只露微小缝隙（无借出记录 / 想不起自己名字等）；夜 C（§12.3）以「被遗忘的书」揭晓身份、可带人贩子假象，但揭晓后故事不收尾，只是继续走的一环；终章落 BE，把「如何与这段遗憾和解」的代理权交还玩家。

**语气姿态**：安静、不说教、偶尔一句戳破。行文参考 §11.1（《去你的岛》式克制、疏离又温柔）。

**固定口癖（已定）：每夜结尾补一句「灯还亮着。」**
- 字面：图书馆的灯为你留着，天亮前这里一直开着。
- 潜层：我在，你随时可以回来。
- **与终极执念的咬合（关键）**：前几夜它是近乎自动的打烊铃，轻、routine；随着羁绊加深（夜 B→C）它渐有分量；到终章玩家拼出那个环时，它变成那句收束——「灯还亮着。因为我是你，那个还没找到自己、所以留下来等你的你。」口癖即伏笔，前期越轻，后期越重；而「灯」的隐喻此时双层落地：既指图书馆的灯，也指「你心里那盏还没灭的、属于自己的灯」。这句收束是暖的、遗憾的、不圆满的——它点明 BE 落点，而非大团圆。
- 配合小动作：每晚说完正事，停一拍，再落「灯还亮着。」（沉默 + 一句，复用 §3 留白即节奏）。

## 16. 风险与开放问题

- **叙事量级**：单夜文字量需控制（建议 1500~3000 字 / 夜），避免冗长。
- **文字质量**：共鸣来自文笔，需专人 / 反复打磨，引擎不解决；§11.1 的《去你的岛》式语气需反复校准，避免「学了个形没学到神」。
- **重复可玩性**：单元剧结构下，重玩动力来自多结局与元层 / 终极执念，需在 M2 验证。
- **多结局去游戏化后是否仍有趣**：移除「失败」标签后，需确保四种结果各自有足够的情绪重量，否则玩家无所适从。
- **夜 B 探索深度**：若要做成「冒险探索」，需定义更丰富的空间互动，可能超出 M1 引擎范围，留 M2 评估。
- **终极执念过早暴露**：这是后期才揭晓的真相，若前期说太满，既泄底又出戏，也破坏「不能让玩家发现」的表演约束。纪律：夜 A–B 管理员只是「人」、只是馆员；只埋微小异常，揭晓严格放在夜 C / 终章、玩家已依恋之后，且以玩家自行拼合的方式呈现，不靠独白直说。
- **BE 沦为「为悲而悲」或作者秀**：元层最强，若揭晓后不给玩家代理权、或把「TA 是你」当抖包袱，易变「作者聪明」而非「玩家共鸣」；若 BE 没被前情赚到，只是强行致郁，会显廉价。对策：BE 必须靠前期碎片（信、人、未完成的事、§2.1 的破碎底色）自然铺满，让意难平是「回不去」而非「被虐」；终章仍把「如何与遗憾和解」的代理权交还玩家。
- **莫比乌斯环的可读性（书≠你，是遗忘的碎片）**：自我循环的隐喻若线索太弱，玩家二周目也拼不出环，会觉「故弄玄虚」；但若把书直接写成「你」，又过度。需精确：书=你遗忘的**一部分**（碎片），馆员=你遗落的**版本**。前夜埋足可回照的微小异常，终章提供清晰的镜像回照，让「哦，原来一直陪我的，是我自己留下的那部分」成立而非强行。
- **投射式装置疲劳（§10 专属）**：若每章都做「暖假结局 → 揭穿投射」翻转，会疲劳、变廉价，且稀释 §2.1 终极真相的冲击。对策：旗舰只用母亲章等 1–2 处重度示范；其余章以「轻度暖假结局 + 留白」为主，控频使用。
- **人贩子假象的类型错位（§12.3 专属）**：夜 C 把管理员误认为人贩子，是有效的悬念 red herring，但若铺垫不当，会让作品滑向悬疑 / 惊悚类型，与整体克制安静的基调冲突。对策：人贩子线索只作「误认感」的氛围手段，揭穿必须快速、干净地落回「自我指认 + 继续走」，不展开追踪 / 对抗情节。
- **「自我早已破碎」的呈现分寸（§2.1 专属）**：这是全作最锋利也最易失控的设定。写得太重会变致郁宣言，写得太轻又立不住 BE。对策：前期不直说，靠碎片与暖假结局的反差自然渗出；终章由玩家拼合，作者只给语境、不给结论。
